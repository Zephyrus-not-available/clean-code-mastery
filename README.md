# 📚 Clean Code Mastery (Java)

မင်္ဂလာပါ။ ဒီ Repository ကနေ ကြိုဆိုပါတယ်။ Robert C. Martin ရေးသားထားတဲ့ **Clean Code** စာအုပ်ရဲ့ အခြေခံမူများကို လေ့လာရလွယ်ကူအောင် ဒီနေရာမှာ စနစ်တကျ ပြင်ဆင်ပေးထားပါတယ်။

Code ရေးသားရာမှာ ကိုယ်ပိုင် ဉာဏ်စွမ်းပြဖို့ထက် တခြားသူ ဖတ်ရှုနားလည်လွယ်ဖို့က ပိုအရေးကြီးပါတယ်။ ဒီ Repository ကနေတစ်ဆင့် Clean Code စည်းမျဉ်းများကို လက်တွေ့ ဥပမာများနှင့်တကွ လေ့လာနိုင်မှာ ဖြစ်ပါတယ်။

## 📂 Repository တွင် ပါဝင်သော ဖိုင်များ (Contents)

ကျောင်းသားများအနေဖြင့် `src/main/java/com/cleancode/` အောက်ရှိ အခန်းတစ်ခုချင်းစီ (ch02_naming, ch03_functions စသည်) တွင် အောက်ပါဖိုင်များကို အစဉ်လိုက် လေ့လာရန် အကြံပြုလိုပါသည် -

### ၁။ `theory.md` (အဓိက သီအိုရီ သင်ခန်းစာများ)
Clean Code စာအုပ်ထဲတွင် ပါဝင်သော အခန်းလိုက် အခြေခံမူများကို ရှင်းလင်းထားသော ဖိုင်ဖြစ်သည်။
- နားလည်ရခက်သော "မကောင်းသော ဥပမာ (Bad)" နှင့် ပြင်ဆင်ထားသည့် "ကောင်းသော ဥပမာ (Good)" များကို နှိုင်းယှဉ်ပြထားသည်။

### ၂။ `solid_review.md` (SOLID Principles နှင့် ဆက်စပ်မှု)
ထိုအခန်းပါ အကြောင်းအရာသည် Object-Oriented Programming ၏ အသက်ဖြစ်သော **SOLID Principles** များနှင့် မည်သို့ ဆက်စပ်နေသနည်း ဆိုသည်ကို ဆန်းစစ်ထားသော မှတ်စုဖြစ်သည်။

### ၃။ `exercise.md` (လက်တွေ့ လေ့ကျင့်ခန်း)
သီအိုရီများကို နားလည်သွားပြီဆိုပါက လက်တွေ့ စမ်းသပ်ရန် ဖြစ်ပါသည်။
- အဓိပ္ပါယ်မရှိသော `Dirty Code` များကို ဖတ်ပြီး `Clean Code` ဖြစ်အောင် ကိုယ်တိုင် ပြင်ဆင်ရေးသားရမည့် လေ့ကျင့်ခန်း ပါဝင်သည်။
- သက်ဆိုင်ရာ အခန်း၏ `dirty/` package အောက်ရှိ Java ကုဒ်များကို ဝင်ရောက်၍ `clean/` package တွင် ရှိသင့်သည့် ပုံစံမျိုးဖြစ်အောင် ပြင်ဆင်ပါ။

### ၄။ `study_methodology.md` (လေ့လာမှု နည်းစနစ်များ)
အခန်းတစ်ခုချင်းစီကို မည်သို့ ထိရောက်စွာ လေ့လာရမည်ဆိုသည့် အဓိက အကြံပြုချက်များနှင့် ချဉ်းကပ်ပုံ နည်းစနစ်များကို ဖော်ပြထားသော ဖိုင်ဖြစ်သည်။

## 💡 လေ့လာရန် အကြံပြုချက် (How to Study)
1. ပထမဦးစွာ `study_methodology.md` ကို ဖတ်ရှုပြီး ထိုအခန်းအတွက် လေ့လာရမည့် နည်းစနစ်ကို နားလည်အောင် လုပ်ပါ။
2. ထို့နောက် `theory.md` ကို အချိန်ယူဖတ်ရှုပါ။ Code ဥပမာတစ်ခုချင်းစီကို သေချာကြည့်ပါ။
3. `solid_review.md` ကို ဖတ်ပြီး Design Pattern များနှင့် ယုတ္တိကို ချိတ်ဆက်ပါ။
4. နောက်ဆုံးတွင် `exercise.md` မှ ညွှန်ကြားချက်များကို ဖတ်ပြီး `dirty` package ထဲက code ကို Clean ဖြစ်အောင် ကိုယ်တိုင် ပြင်ရေးကြည့်ပါ။
5. Unit tests များ (`src/test/...`) ပါဝင်ပါက ကုဒ်ပြင်ပြီးနောက် Test များ အောင်မြင်မှု ရှိမရှိ run ကြည့်ပါ။

အားလုံးပဲ Clean Coder ကောင်းများ ဖြစ်လာကြပါစေ။
