# SOLID Review - Chapter 6 Objects and Data Structures

- [ ] **S (Single Responsibility):** Are data and behavior responsibilities intentionally separated?
- [ ] **O (Open/Closed):** Is extension handled through polymorphism where appropriate?
- [ ] **L (Liskov Substitution):** Do object hierarchies preserve behavior invariants?
- [ ] **I (Interface Segregation):** Do data consumers rely only on required operations?
- [ ] **D (Dependency Inversion):** Do policies consume object abstractions, not DTO internals?
