# SOLID Review - Chapter 7 Error Handling

- [ ] **S (Single Responsibility):** Is error handling separated from business logic?
- [ ] **O (Open/Closed):** Can new failure modes be added without rewriting core paths?
- [ ] **L (Liskov Substitution):** Do subtype failures preserve parent expectations?
- [ ] **I (Interface Segregation):** Do clients avoid broad exception-heavy contracts?
- [ ] **D (Dependency Inversion):** Do high-level flows depend on error abstractions, not transport details?
