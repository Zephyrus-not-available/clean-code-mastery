# SOLID Review - Chapter 9 Unit Tests

- [ ] **S (Single Responsibility):** Does each test verify one behavior?
- [ ] **O (Open/Closed):** Do tests support adding features without brittle rewrites?
- [ ] **L (Liskov Substitution):** Do contract tests validate substitutability?
- [ ] **I (Interface Segregation):** Are test doubles scoped to client needs?
- [ ] **D (Dependency Inversion):** Are high-level tests using abstractions and seams?
