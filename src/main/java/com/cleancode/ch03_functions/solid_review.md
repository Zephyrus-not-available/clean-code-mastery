# SOLID Review - Chapter 3 Functions

- [ ] **S (Single Responsibility):** Does each function do one thing at one level of abstraction?
- [ ] **O (Open/Closed):** Can I add a new behavior path without editing a monolithic function?
- [ ] **L (Liskov Substitution):** Do function contracts remain valid for all subtypes?
- [ ] **I (Interface Segregation):** Are function-level interfaces focused and minimal?
- [ ] **D (Dependency Inversion):** Do functions depend on abstractions rather than concrete services?
