# SOLID Review - Chapter 10 Classes

- [ ] **S (Single Responsibility):** Is each class cohesive and change-focused?
- [ ] **O (Open/Closed):** Can behavior be extended with composition/polymorphism?
- [ ] **L (Liskov Substitution):** Do subclass invariants preserve parent guarantees?
- [ ] **I (Interface Segregation):** Are class APIs split by client roles?
- [ ] **D (Dependency Inversion):** Are class dependencies inverted toward abstractions?
